Green Software Carbon Toolkit is a state-of-the-art, web-based platform designed to help developers, architects, and organizations quantify and mitigate the environmental impact of their digital products. By transforming runtime performance metrics into actionable climate-aware insights, it bridges the gap between software engineering and environmental responsibility.

## ✨ Key Features

- **📊 Intelligent Carbon Scoring**: Calculate energy consumption and carbon intensity for various project architectures (Frontend, Backend, ML, Mobile, etc.).
- **☁️ Cloud Impact Analysis**: Evaluate the carbon footprint across major cloud providers (AWS, GCP, Azure) using regional carbon intensity data.
- **🔍 Explainability Panel**: Deep dive into the factors influencing your score, from hardware efficiency to data transfer costs.
- **📈 Comparative Benchmarking**: Visualize how your project stacks up against industry standards and sustainability targets.
- **💡 Actionable Sustainability Tips**: Receive personalized recommendations to optimize your code for both performance and the planet.
- **🛡️ Privacy-First Design**: Uses industry-standard benchmarks and estimation methods rather than invasive device monitoring.

## 🛠️ Tech Stack

### Frontend
- **Framework**: [React 19](https://react.dev/) (Vite)
- **Routing**: [React Router 7](https://reactrouter.com/)
- **Visualizations**: Dynamic Charts & Data Visualization
- **Styling**: Modern, responsive CSS with glassmorphism and animations.

### Backend
- **Runtime**: [Node.js](https://nodejs.org/)
- **Framework**: [Express.js](https://expressjs.com/)
- **Security**: CORS enabled for secure cross-origin communication.

## 🚀 Getting Started

### Prerequisites
- Node.js (v18 or higher)
- npm or yarn

### Installation

1. **Clone the repository**:
   ```bash
   git clone https://github.com/Sm-LumeCode/green-software-carbon-toolkit.git
   ```

2. **Setup Backend**:
   ```bash
   cd backend
   npm install
   npm start
   ```

3. **Setup Frontend**:
   ```bash
   cd ../greenscore
   npm install
   npm run dev
   ```

## 🌍 Why Greenscore?

"Every commit has a cost. Greenscore helps you see it."

Applications are scaling faster than responsibility. Servers expand, clouds multiply, and memory grows silently—yet carbon metrics rarely appear in a terminal. Greenscore makes this hidden cost visible, allowing developers to make informed decisions that balance performance with sustainability.
